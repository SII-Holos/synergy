# Plugin API Migration

This note is for plugins created before the current descriptor, manifest, and packaged-artifact contract. New plugins should use the [plugin getting-started guide](../plugins/getting-started.md).

## Use an Object Descriptor

Export one `PluginDescriptor` whose ID is the canonical plugin ID:

```ts
import type { PluginDescriptor } from "@ericsanchezok/synergy-plugin"

const plugin: PluginDescriptor = {
  id: "my-plugin",
  async init(input) {
    return {
      tool: {},
    }
  },
}

export default plugin
```

Set `plugin.json.name` to the same value. Configuration, credentials, cache, approval, lockfile, signature, and registry records use that identity.

## Make the Manifest Complete

Add `name`, semantic `version`, `description`, compatibility, and the source runtime entry. Declare every runtime tool in `contributes.tools`, including exposure, display metadata, and capabilities. Add explicit plugin-wide permissions for every tool service, data class, network domain, UI surface, delegated task, or hook the plugin uses.

Runtime discovery now rejects undeclared tools and meaningful capability mismatches.

## Build UI Assets

Solid UI source belongs in `src/ui.tsx`, `src/ui/index.tsx`, or the equivalent TypeScript entry recognized by the plugin kit. The manifest points to the distributable JavaScript asset, normally:

```jsonc
{
  "permissions": { "ui": true },
  "contributes": {
    "ui": {
      "entry": "./ui/index.js",
      "minUIApiVersion": "3.0",
    },
  },
}
```

Use the UI API value generated by the installed kit. The build externalizes the shared Solid runtime and writes the JS entry into `dist/`.

## Validate and Repackage

```bash
synergy-plugin validate --runtime-discovery
synergy-plugin build
synergy-plugin pack
```

Fix identity mismatch, undeclared contributions, capability warnings, missing exports, and unsafe asset paths before publishing. Install the generated `.synergy-plugin.tgz`; a repository source archive is not an installable release artifact.

## Reapprove Changed Permissions

Approval is bound to the plugin ID and manifest/permissions hashes. A migrated or expanded manifest requires a new review. Do not suppress the diff by retaining an incomplete old declaration.
