Update an existing blueprint note. Content should be provided in markdown format — it will be converted to the internal ProseMirror JSON format for storage.

Two modes are available:
- "append": Adds content to the end of an existing blueprint. Requires `id` and `content`. The new content is appended as additional blocks after the existing document's blocks.
- "replace": Overwrites the content of an existing blueprint. Requires `id` and `content`.

Blueprint metadata fields (title, description, status, defaultAgent, tags) can be updated alongside content. The blueprint is found automatically regardless of scope.

Note: For surgical edits to specific parts of a blueprint without rewriting the whole document, use `note_edit` instead.
