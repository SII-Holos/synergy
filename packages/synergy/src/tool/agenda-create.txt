Create a scheduled task, recurring job, or reminder executed by an agent.

Use this for RECURRING schedules: daily reviews, periodic reminders, cron jobs. The task runs a prompt in a new session each time it fires.

If you need to WAIT for a condition (e.g. an experiment to finish), use `agenda_watch` instead — it's simpler, self-terminating, and doesn't create sessions.

The `prompt` is the instruction the agent follows when the item fires. Write it as if you're briefing another agent on what to do.

Items with triggers (cron, every, at, delay) are automatically activated. Items without triggers start as "pending" — use `agenda_trigger` to run them manually.

By default, when execution completes, the result is delivered back to the originating session and wakes the agent (wake=true). Set `silent: true` to suppress delivery, or `wake: false` to deliver without waking the agent.

Items are scoped to the current project by default. Set `global: true` to make an item visible from all projects.

Related tools:
- `agenda_watch` — wait for a condition to be met (self-terminating, no session creation)
- `agenda_list` — see all agenda items
- `agenda_cancel` — stop or remove an item
- `agenda_trigger` — manually trigger an item
- `agenda_logs` — view execution history
