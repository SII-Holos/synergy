Create an agenda item — a scheduled task, recurring job, reminder, or todo that will be executed by an agent.

The `prompt` is the instruction the agent follows when the item fires. Write it as if you're briefing another agent on what to do. If the task needs context from prior conversations, use `sessionRefs` to attach relevant sessions with a `hint` explaining what to focus on in each.

By default, items execute in the global (home) scope. To target a specific project, provide `workScopeID` or `workDirectory`. Use `session_list` with scope "project" to discover available scope IDs and directories.

Items with time-based triggers (cron, every, at, delay) are automatically activated. Items without triggers start as "pending" — use `agenda_trigger` to run them manually, or `agenda_update` to add triggers later.

The default `delivery` is "auto" — results are delivered back to the current session (the one where the user made the request). In most cases, omit `delivery` entirely. "auto" adapts to the originating context: in a Feishu group chat it sends results back to that group, in a DM back to that DM, in the web app back to that session. Only override when needed: "home" forces delivery to the app home session, "silent" suppresses delivery entirely, and "session" targets a specific session (requires `deliverySessionID`). Do not set `delivery` to "home" unless the user explicitly asks or the task has no meaningful originating context.

When `delivery` is "session", you must also provide `deliverySessionID`. Use `session_list` to find the target session.
