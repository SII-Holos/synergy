Use this from the active Blueprint audit session when the audit finds remaining work.

This sends a restart message to the original execution session, records the audit reason, and returns the loop to running.

Parameters:
- loopID: The BlueprintLoop ID under audit
- reason: Clear explanation of why the loop is not complete
- completed: Optional summary of work that is already correct
- remaining: Optional summary of missing or incorrect work
- instructions: Optional next actions for the execution agent
