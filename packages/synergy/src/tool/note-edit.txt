Performs precise block-level edits to a note's content. This includes Blueprint notes, because a Blueprint is a note with kind:"blueprint".

This tool operates on numbered blocks from the note's internal document model (ProseMirror JSON). Each block corresponds to a top-level element: a paragraph, heading, list, code block, image, etc. The block index is 1:1 with `note_read` format:'blocks' output.

Usage:
- You MUST first call `note_read({ ids: [...], format: "blocks" })` to see the numbered block listing. Use those block indices in the `ops` array.
- Operations are applied in descending index order automatically, so the indices you see in the blocks output are always correct regardless of operation order.
- `replace`: overwrite the block at the given index with new markdown content. The markdown is parsed into one or more blocks.
- `insertAfter`: insert new markdown content AFTER the block at the given index.
- `delete`: remove the block at the given index (no `content` needed).
- All operations require `content` except `delete`.
- Blocks NOT referenced in any operation (e.g., Mermaid diagrams, videos, images, tables) are preserved intact. Only explicitly targeted blocks are modified.
- Empty documents have 0 blocks. Use `note_write` mode:"append" or mode:"replace" to add content to an empty note, not `insertAfter`.
- Version conflict: if the note was modified by another process between your read and edit, the tool returns a conflict error.

Contrast with `note_write`:
- `note_write` mode:"replace" overwrites the ENTIRE document — use for full rewrites.
- `note_write` mode:"append" adds content to the END — use for adding new sections.
- `note_edit` performs surgical block-level edits — use for fixing typos, updating details, inserting between existing blocks, or deleting specific sections without touching the rest.
