Submit exactly one semantic Lattice action for the current state. Valid actions are: submit_requirements in clarifying; submit_pathway in planning; submit_pathway_review in reviewing_pathway; submit_blueprint in blueprinting; submit_blueprint_review in reviewing_blueprint; and approve_execution in awaiting_execution. The root input is strict: include every field required by the selected action and no fields belonging to another action. Submission durably queues the action with current state and Pathway revisions; the Lattice controller validates and advances it. Repeating the same accepted action is idempotent, while a different pending action or stale state is a conflict.
