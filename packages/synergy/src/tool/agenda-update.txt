Update an existing agenda item. Only provided fields are changed — omitted fields remain unchanged.

Use `agenda_list` to find the item ID first.

Status changes control execution behavior:
- "active": Enables automatic execution based on triggers
- "paused": Suspends execution (re-activate with "active")
- "done": Marks as completed
- "cancelled": Marks as cancelled

To deactivate an item without deleting it, set status to "paused" or "done" rather than using `agenda_delete`.

Updating `triggers` recalculates the next scheduled run time. Updating `prompt` modifies the execution instruction while preserving other task configuration (workScope, sessionRefs).
