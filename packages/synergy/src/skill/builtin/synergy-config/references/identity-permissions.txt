# Engram and Permissions

## Engram — Memory, Experience, and Autonomy

The `engram` section configures the memory/knowledge system.

### Embedding and rerank config

These live at the top level of config:

```jsonc
{
  "embedding": {
    "baseURL": "https://api.siliconflow.cn/v1",
    "apiKey": "{env:SILICONFLOW_API_KEY}",
    "model": "Qwen/Qwen3-Embedding-8B"
  },
  "rerank": {
    "baseURL": "https://api.siliconflow.cn/v1",
    "apiKey": "{env:SILICONFLOW_API_KEY}",
    "model": "Qwen/Qwen3-Reranker-8B"
  }
}
```

| Field | Description |
|---|---:|
| `embedding.baseURL` | Base URL for embedding API |
| `embedding.apiKey` | API key (supports `{env:VAR}`) |
| `embedding.model` | Embedding model name |
| `rerank.baseURL` | Base URL for rerank API |
| `rerank.apiKey` | Rerank API key (falls back to `embedding.apiKey`) |
| `rerank.model` | Rerank model name |

### Engram config

```jsonc
{
  "engram": {
    "memory": {
      "enabled": true,
      "simThreshold": 0.7,
      "topK": 8,
      "dedupThreshold": 0.75
    },
    "experience": {
      "encode": true,
      "retrieve": true
    },
    "autonomy": true
  }
}
```

| Field | Description |
|---|---:|
| `engram.memory.enabled` | Enable/disable memory subsystem |
| `engram.memory.simThreshold` | Minimum cosine similarity for retrieval candidates (default: 0.7) |
| `engram.memory.topK` | Number of memories to retrieve (default: 8) |
| `engram.memory.dedupThreshold` | Similarity threshold for deduplication (default: 0.75) |
| `engram.experience.encode` | Learn from conversations — extract intent, reward, and scripts |
| `engram.experience.retrieve` | Inject relevant past experiences into new conversations |
| `engram.autonomy` | Enable autonomous background routines like anima (default: true) |

### Experience learning config (advanced)

```jsonc
{
  "engram": {
    "experience": {
      "encode": true,
      "retrieve": true,
      "learning": {
        "alpha": 0.3,
        "qInit": 0.5,
        "dedupIntentThreshold": 0.85,
        "dedupScriptThreshold": 0.8,
        "qHistorySize": 50,
        "snapThreshold": 0.5,
        "rewardDelay": 5,
        "rewardWeights": {
          "outcome": 0.35,
          "intent": 0.25,
          "execution": 0.2,
          "orchestration": 0.1,
          "expression": 0.1
        }
      }
    }
  }
}
```

### Memory retrieval config (advanced)

```jsonc
{
  "engram": {
    "memory": {
      "enabled": true,
      "simThreshold": 0.5,
      "topK": 5,
      "dedupThreshold": 0.75,
      "categories": {
        "interaction": { "simThreshold": 0.4, "topK": 6 },
        "coding": { "simThreshold": 0.55, "topK": 8 },
        "knowledge": { "simThreshold": 0.6, "topK": 10 }
      }
    }
  }
}
```

Memory categories: `user`, `self`, `relationship`, `interaction`, `workflow`, `coding`, `writing`, `asset`, `insight`, `knowledge`, `personal`, `general`.

Memory retrieval is driven per row by `recallMode`: `always`, `contextual`, or `search_only`. Use `always` sparingly for durable identity and interaction rules rather than assuming a whole category is always loaded.

### CLI commands

```
synergy config embedding  — Interactive configure embedding provider
synergy config rerank     — Interactive configure rerank provider
synergy engram show       — Display engram config
synergy engram learning   — Configure experience learning parameters
synergy engram memory     — Configure memory parameters
```

## Permissions

Permissions control tool access: allow, deny, or require approval.

### Permission actions

| Action | Meaning |
|---|---:|
| `"allow"` | Execute without user approval |
| `"ask"` | User must approve each invocation |
| `"deny"` | Blocked entirely |

### Top-level permissions (all agents)

```jsonc
{
  "permission": {
    "read": "allow",
    "edit": "allow",
    "glob": "allow",
    "grep": "allow",
    "list": "allow",
    "bash": "ask",
    "task": "allow",
    "webfetch": "allow",
    "websearch": "allow",
    "external_directory": "deny",
    "download": "allow",
    "lsp": "allow",
    "doom_loop": "allow",
    "dagwrite": "allow",
    "dagread": "allow",
    "question": "allow",
    "todowrite": "allow",
    "todoread": "allow",
    "skill": "allow"
  }
}
```

### Per-agent permissions

Override in agent config:
```jsonc
{
  "agent": {
    "safe-agent": {
      "permission": { "bash": "deny", "edit": "deny" }
    }
  }
}
```

Agent permissions merge with and override top-level permissions.

### Path-scoped permissions

Some tools support pattern-based rules:

```jsonc
{
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "npm test": "allow",
      "rm *": "deny"
    },
    "edit": {
      "*": "ask",
      "src/**": "allow",
      "*.config.*": "deny"
    },
    "read": {
      "*": "allow",
      ".env*": "deny"
    }
  }
}
```

`*` is the default rule. More specific patterns override it. Rules are evaluated in order — first matching rule wins.

### Shorthand

Top-level shorthand for all tools:
```jsonc
{ "permission": "allow" }
```

Equivalent to setting all tools to `"allow"`.

### Common permission patterns

**Read-only agent** — can read and search but never modify:

```jsonc
{ "permission": { "edit": "deny", "write": "deny", "bash": "deny" } }
```

**Sandboxed execution** — bash allowed only for specific safe commands:

```jsonc
{ "permission": { "bash": { "*": "ask", "git status": "allow", "npm test": "allow", "bun test": "allow" } } }
```

**Protect sensitive files** — read everything except secrets:

```jsonc
{ "permission": { "read": { "*": "allow", ".env*": "deny", "*credentials*": "deny", "*.pem": "deny" } } }
```

**Full trust** — allow everything without asking:

```jsonc
{ "permission": "allow" }
```

**Source code only** — only edit source files, not config:

```jsonc
{ "permission": { "edit": { "*": "ask", "src/**": "allow", "test/**": "allow" }, "write": { "*": "ask", "src/**": "allow" } } }
```

### Troubleshooting

- **Agent getting "permission denied"** — check both top-level and agent-level permissions; agent-level overrides top-level; check path-scoped rules (more specific patterns override `*`)
- **Permission change not taking effect** — reload with `runtime_reload(target: "config")`; permission changes cascade to agent reload automatically
- **"ask" prompts too frequent** — consider path-scoped rules to auto-allow safe patterns; use `*` as default with specific overrides
- **Memory/engram not working** — check `engram.autonomy` is not `false`; check `engram.memory.enabled` is not `false`; check `engram.experience.encode` / `engram.experience.retrieve` are not `false`
- **Embedding/rerank errors** — verify API key and baseURL in top-level `embedding` / `rerank` sections; check the model name matches what the provider expects; embedding failures are logged at startup

## Reload

```
runtime_reload(target: "config")
```

Engram and permission changes cascade to agent reload automatically.
