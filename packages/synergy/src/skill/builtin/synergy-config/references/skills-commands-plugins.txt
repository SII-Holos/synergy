# Skills, Commands, and Plugins

## Skills

Skills extend Synergy with specialized knowledge, workflows, and tool integrations. They are modular markdown files that teach Synergy domain-specific procedures.

### Skill file format

Every skill is a `SKILL.md` file with YAML frontmatter:

```markdown
---
name: my-skill
description: "Description of when to use this skill. Include triggers and use cases."
---

# My Skill

Instructions and guidance for using the skill.
```

**Required frontmatter:** `name`, `description`.

The `description` is critical — it's the only text Synergy sees when deciding whether to load the skill. Be comprehensive about what the skill does and when to trigger it.

The body is only loaded AFTER the skill triggers, so "When to Use" sections in the body are wasted.

### Skill directories

Skills are discovered from these directories:

| Scope | Path pattern |
|---|---|
| Project | `<project>/.synergy/{skill,skills}/**/SKILL.md` |
| Global | `~/.synergy/config/{skill,skills}/**/SKILL.md` |
| Claude compat (project) | `<project>/.claude/skills/**/SKILL.md` |
| Claude compat (global) | `~/.claude/skills/**/SKILL.md` |

Example layout:
```
.synergy/skill/
├── my-skill/
│   ├── SKILL.md
│   ├── references/
│   │   └── api-docs.md
│   ├── scripts/
│   │   └── helper.py
│   └── assets/
│       └── template.html
```

### Progressive disclosure

Skills use three-level loading to manage context:

1. **Metadata** (name + description) — always in context (~100 words)
2. **SKILL.md body** — loaded when skill triggers (keep under 500 lines)
3. **Bundled resources** — loaded by agent as needed (unlimited)

When SKILL.md approaches 500 lines, split detailed content into `references/` files and reference them clearly:
```markdown
For API details, see references/api-docs.md.
For conversion rules, see references/format-rules.md.
```

### Skill resources

- **`references/`** — documentation loaded into context as needed. Keep SKILL.md lean.
- **`scripts/`** — executable code (Python/Bash) for deterministic tasks. Token-efficient.
- **`assets/`** — files used in output (templates, images). Not loaded into context.

### Reload

```
runtime_reload(target: "skill")
```

File watcher also auto-detects SKILL.md changes.

## Commands

Custom commands are reusable prompt templates triggered via `/command-name` in TUI/Web.

### Command file format

Create `.synergy/command/<name>.md` (project) or `~/.synergy/config/command/<name>.md` (global):

```markdown
---
description: "Run code review on staged changes"
agent: "synergy"
model: "anthropic/claude-sonnet-4-5"
---

Review the staged git changes. Focus on:
- Security issues
- Performance concerns
- Code style violations

Provide a summary with severity ratings.
```

**Frontmatter fields:**
- `description` — shown in command list UI
- `agent` — optional: which agent handles this command
- `model` — optional: model override for this command

**Body** = the prompt template that gets injected.

### Command directories

| Scope | Path pattern |
|---|---|
| Project | `<project>/.synergy/{command,commands}/**/*.md` |
| Global | `~/.synergy/config/{command,commands}/**/*.md` |

Nested commands: `.synergy/command/review/security.md` → `/review/security`

### Reload

```
runtime_reload(target: "command")
```

## Plugins

Plugins are TypeScript/JavaScript files that extend Synergy's tool and provider capabilities.

### Plugin file locations

| Scope | Path pattern |
|---|---|
| Project | `<project>/.synergy/{plugin,plugins}/*.{ts,js}` |
| Global | `~/.synergy/config/{plugin,plugins}/*.{ts,js}` |

### Plugin API

Plugins use the `@ericsanchezok/synergy-plugin` SDK. A plugin file exports a default function:

```ts
import { definePlugin } from "@ericsanchezok/synergy-plugin"

export default definePlugin({
  name: "my-plugin",
  tools: [
    // custom tool definitions
  ],
  auth: {
    provider: "my-provider",
    // optional auth loader for custom provider authentication
  },
})
```

Synergy auto-installs the plugin SDK (`@ericsanchezok/synergy-plugin`) in each config directory that has plugins.

### Plugin dependencies

Plugins can use external npm packages. Add them to the config directory's `package.json`:
```bash
cd .synergy && bun add some-package
```

### Reload

```
runtime_reload(target: "plugin")
```

Plugin reload cascades to tool_registry reload.
