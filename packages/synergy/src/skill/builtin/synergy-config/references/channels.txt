# Channels

## Channels

Channels integrate Synergy with external messaging platforms.

### Setting up Feishu/Lark ‚Äî step by step

1. **Create a Feishu app** ‚Äî Go to [Feishu Open Platform](https://open.feishu.cn/app) (China) or [Lark Developer](https://open.larksuite.com/app) (international). Create a new app and note the App ID and App Secret.
2. **Enable bot capability** ‚Äî In the app settings, enable "Robot" capability under "Application Capabilities".
3. **Configure event subscription** ‚Äî Set the request URL to your Synergy server's public endpoint: `https://<your-server>/api/channel/feishu/webhook`. Enable events: `im.message.receive_v1`.
4. **Add permissions** ‚Äî Grant these scopes: `im:message`, `im:message:send_as_bot`, `im:chat:readonly`, `im:resource`. For sender name resolution: `contact:user.base:readonly`.
5. **Publish the app** ‚Äî Submit for review or use in development mode for testing.
6. **Configure Synergy** ‚Äî Add the channel config to `90-channels.jsonc` with your App ID and App Secret.
7. **Reload** ‚Äî `runtime_reload(target: "channel")`.

### Feishu/Lark channel

```jsonc
{
  "channel": {
    "my-feishu": {
      "type": "feishu",
      "domain": "feishu",
      "streaming": true,
      "accounts": {
        "main": {
          "enabled": true,
          "appId": "cli_xxx",
          "appSecret": "xxx",
          "domain": "feishu",
          "allowDM": true,
          "allowGroup": true,
          "requireMention": true,
          "streaming": true,
          "streamingThrottleMs": 100,
          "groupSessionScope": "group",
          "inboundDebounceMs": 0,
          "resolveSenderNames": true,
          "replyInThread": false
        }
      }
    }
  }
}
```

| Field | Default | Description |
|---|---|---|
| `appId` | (required) | Feishu app ID |
| `appSecret` | (required) | Feishu app secret |
| `domain` | auto | `"feishu"` (China) or `"lark"` (international) |
| `enabled` | true | Enable this account |
| `allowDM` | true | Allow direct messages |
| `allowGroup` | true | Allow group messages |
| `requireMention` | true | Require @mention in group chats |
| `streaming` | true | Enable streaming card updates |
| `streamingThrottleMs` | 100 | Min interval between streaming updates (ms) |
| `groupSessionScope` | `"group"` | Session scoping strategy (see below) |
| `inboundDebounceMs` | 0 | Debounce rapid-fire messages (0 = disabled) |
| `resolveSenderNames` | true | Resolve sender names via Feishu contact API |
| `replyInThread` | false | Reply in thread when message is part of a topic |

**Group session scoping:**
- `"group"` ‚Äî one shared session per group chat
- `"group_sender"` ‚Äî separate session per sender in group
- `"group_topic"` ‚Äî separate session per thread/topic
- `"group_topic_sender"` ‚Äî separate session per thread + sender

Top-level `domain` and `streaming` on the channel object set defaults for all accounts.

### Holos channel

```jsonc
{
  "channel": {
    "holos": {
      "type": "holos",
      "apiUrl": "https://www.holosai.io",
      "wsUrl": "wss://www.holosai.io",
      "portalUrl": "https://www.holosai.io",
      "accounts": {
        "default": { "enabled": true }
      }
    }
  }
}
```

### Reload

```
runtime_reload(target: "channel")
```

## Domain file

Channel config belongs only in:

```
~/.synergy/config/synergy.d/90-channels.jsonc
<project>/.synergy/synergy.d/90-channels.jsonc
```

Do not edit model, provider, email, Holos, or runtime domain files when importing or updating channels.

## Troubleshooting

### Feishu/Lark

- **Bot not responding in group chat** ‚Äî check `requireMention` (default true); the bot needs to be @mentioned unless `requireMention: false`; verify the bot is added to the group; check `allowGroup` is true
- **Bot not responding in DM** ‚Äî check `allowDM` is true; verify the user has permission to message the bot
- **Streaming not working** ‚Äî check `streaming` is true; some older Feishu versions may not support streaming cards; try `streaming: false` for compatibility
- **Sender names showing as IDs** ‚Äî set `resolveSenderNames: true` and ensure the app has `contact:user.base:readonly` scope
- **Webhook not receiving events** ‚Äî verify the Feishu app's event subscription URL is correct and publicly accessible; check the server is running and the `/api/channel/feishu/webhook` endpoint responds
