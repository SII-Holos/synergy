# Channels

## Channels

Channels integrate Synergy with external messaging platforms.

### Setting up Feishu/Lark — step by step

1. **Create a Feishu app** — Go to [Feishu Open Platform](https://open.feishu.cn/app) (China) or [Lark Developer](https://open.larksuite.com/app) (international). Create a new app and note the App ID and App Secret.
2. **Enable bot capability** — In the app settings, enable "Robot" capability under "Application Capabilities".
3. **Configure event subscription** — Set the request URL to your Synergy server's public endpoint: `https://<your-server>/api/channel/feishu/webhook`. Enable events: `im.message.receive_v1`.
4. **Add permissions** — Grant these scopes: `im:message`, `im:message:send_as_bot`, `im:chat:readonly`, `im:resource`. For sender name resolution: `contact:user.base:readonly`.
5. **Publish the app** — Submit for review or use in development mode for testing.
6. **Configure Synergy** — Add the channel config to `90-channels.jsonc` with your App ID and App Secret.
7. **Reload** — `runtime_reload(target: "channel")`.

### Feishu/Lark channel

```jsonc
{
  "channel": {
    "my-feishu": {
      "type": "feishu",
      "domain": "feishu",
      "streaming": true,
      "accounts": {
        "main": {
          "enabled": true,
          "appId": "cli_xxx",
          "appSecret": "xxx",
          "domain": "feishu",
          "allowDM": true,
          "allowGroup": true,
          "requireMention": true,
          "streaming": true,
          "streamingThrottleMs": 100,
          "groupSessionScope": "group",
          "inboundDebounceMs": 0,
          "model": "openai/gpt-5",
          "variant": "high",
          "resolveSenderNames": true,
          "replyInThread": false
        }
      }
    }
  }
}
```

| Field | Default | Description |
|---|---|---|
| `appId` | (required) | Feishu app ID |
| `appSecret` | (required) | Feishu app secret |
| `domain` | auto | `"feishu"` (China) or `"lark"` (international) |
| `botOpenId` | (none) | Bot open_id used to verify real @mentions in group chats |
| `model` | (none) | Model override in `providerID/modelID` format (e.g. `openai/gpt-4o`) |
| `variant` | (none) | Variant exposed by the selected account model (e.g. `low`, `high`, `max`) |
| `enabled` | true | Enable this account |
| `allowDM` | true | Allow direct messages |
| `allowGroup` | true | Allow group messages |
| `requireMention` | true | Require @mention in group chats |
| `streaming` | true | Enable streaming card updates |
| `streamingThrottleMs` | 100 | Min interval between streaming updates (ms) |
| `groupSessionScope` | `"group"` | Session scoping strategy (see below) |
| `inboundDebounceMs` | 0 | Debounce rapid-fire messages (0 = disabled) |
| `resolveSenderNames` | true | Resolve sender names via Feishu contact API |
| `replyInThread` | false | Reply in thread when message is part of a topic |

**Group session scoping:**
- `"group"` — one shared session per group chat
- `"group_sender"` — separate session per sender in group
- `"group_topic"` — separate session per thread/topic
- `"group_topic_sender"` — separate session per thread + sender

Top-level `domain` and `streaming` on the channel object set defaults for all accounts.

### Clarus channel

Clarus accounts reuse the active Holos credential. A successful Holos login adds the matching account when it is absent, so access tokens never belong in Channel config. Explicit account settings are preserved.

```jsonc
{
  "channel": {
    "clarus": {
      "type": "clarus",
      "accounts": {
        "<holos-agent-id>": {
          "enabled": true,
          "agent": "synergy"
        }
      }
    }
  }
}
```

`apiUrl` is an optional per-account override. Clarus has no top-level config domain or workspace-root setting: each remote project is mapped to a standard Synergy Project Scope by the Channel runtime.

### Holos platform configuration

Holos is configured under `100-holos.jsonc`, not in channels:

```jsonc
{
  "holos": {
    "enabled": true,
    "apiUrl": "https://api.holosai.io",
    "wsUrl": "wss://api.holosai.io",
    "portalUrl": "https://www.holosai.io"
  }
}
```

See the Holos domain file (`100-holos.jsonc`) — do not add `type: "holos"` under `channel`.

### Reload

```
runtime_reload(target: "channel")
```

## Domain file

Channel config belongs only in:

```
~/.synergy/config/synergy.d/90-channels.jsonc
<project>/.synergy/synergy.d/90-channels.jsonc
```

Holos config belongs in `100-holos.jsonc`, not in channels. Do not edit model, provider, email, or runtime domain files when importing or updating channels.

## Troubleshooting

### Clarus

- **Account is absent** — sign in to Holos, which creates the matching `channel.clarus` account without copying credentials.
- **Project is unavailable** — verify the Holos account can manage the project; project discovery failures leave existing Scopes and Sessions intact while Channel reconnects.

### Feishu/Lark

- **Bot not responding in group chat** — check `requireMention` (default true); the bot needs to be @mentioned unless `requireMention: false`; verify the bot is added to the group; check `allowGroup` is true
- **Bot not responding in DM** — check `allowDM` is true; verify the user has permission to message the bot
- **Streaming not working** — check `streaming` is true; clients without streaming-card support can use `streaming: false`
- **Sender names showing as IDs** — set `resolveSenderNames: true` and ensure the app has `contact:user.base:readonly` scope
- **Account model ignored** — verify `model` uses `providerID/modelID`; a conversation-level `/model` command overrides the account default until a new conversation is started
- **Effort selector missing** — the selected account model must expose variants in the active provider catalog; models without variants use provider defaults
- **Webhook not receiving events** — verify the Feishu app's event subscription URL is correct and publicly accessible; check the server is running and the `/api/channel/feishu/webhook` endpoint responds
