# Library and Permissions

## Library — Memory, Experience, and Autonomy

The `library` section configures the memory/knowledge system.

### Embedding and rerank config

These are top-level fields in `00-general.jsonc`, not library-specific config:

```jsonc
{
  "embedding": {
    "baseURL": "https://api.siliconflow.cn/v1",
    "apiKey": "{env:SILICONFLOW_API_KEY}",
    "model": "Qwen/Qwen3-Embedding-8B"
  },
  "rerank": {
    "baseURL": "https://api.siliconflow.cn/v1",
    "apiKey": "{env:SILICONFLOW_API_KEY}",
    "model": "Qwen/Qwen3-Reranker-8B"
  }
}
```

| Field | Description |
|---|---:|
| `embedding.baseURL` | Base URL for embedding API |
| `embedding.apiKey` | API key (supports `{env:VAR}`) |
| `embedding.model` | Embedding model name |
| `rerank.baseURL` | Base URL for rerank API |
| `rerank.apiKey` | Rerank API key (falls back to `embedding.apiKey`) |
| `rerank.model` | Rerank model name |

### Library config

```jsonc
{
  "library": {
    "memory": {
      "enabled": true,
      "retrieval": {
        "simThreshold": 0.7,
        "topK": 3
      },
      "dedup": {
        "threshold": 0.75
      }
    },
    "experience": {
      "encode": true,
      "retrieve": true
    },
    "autonomy": true
  }
}
```

| Field | Description |
|---|---:|
| `library.memory.enabled` | Enable/disable memory subsystem |
| `library.memory.retrieval.simThreshold` | Minimum cosine similarity for retrieval candidates (default: 0.7) |
| `library.memory.retrieval.topK` | Number of memories to retrieve per category (default: 3) |
| `library.memory.dedup.threshold` | Similarity threshold for deduplication (default: 0.75) |
| `library.experience.encode` | Learn from conversations — extract intent, reward, and scripts |
| `library.experience.retrieve` | Boolean or a passive retrieval object with `simThreshold`, `topK`, `epsilon`, `wSim`, `wQ`, `explorationConstant`. When `true`, uses defaults. |
| `library.autonomy` | Enable autonomous background routines like anima (default: true) |

### Experience learning config (advanced)

```jsonc
{
  "library": {
    "experience": {
      "encode": true,
      "retrieve": true,
      "learning": {
        "alpha": 0.3,
        "qInit": 0.5,
        "dedupIntentThreshold": 0.85,
        "dedupScriptThreshold": 0.8,
        "qHistorySize": 50,
        "snapThreshold": 0.5,
        "rewardDelay": 5,
        "rewardWeights": {
          "outcome": 0.35,
          "intent": 0.25,
          "execution": 0.2,
          "orchestration": 0.1,
          "expression": 0.1
        }
      }
    }
  }
}
```

### Memory retrieval config (advanced)

```jsonc
{
  "library": {
    "memory": {
      "enabled": true,
      "retrieval": {
        "simThreshold": 0.5,
        "topK": 5,
        "categories": {
          "interaction": { "simThreshold": 0.4, "topK": 6 },
          "coding": { "simThreshold": 0.55, "topK": 8 },
          "knowledge": { "simThreshold": 0.6, "topK": 10 }
        }
      },
      "dedup": {
        "threshold": 0.75
      }
    }
  }
}
```

Memory categories: `user`, `self`, `relationship`, `interaction`, `workflow`, `coding`, `writing`, `asset`, `insight`, `knowledge`, `personal`, `general`.

Memory retrieval is driven per row by `recallMode`: `always`, `contextual`, or `search_only`. Use `always` sparingly for durable identity and interaction rules.

```
synergy library memory     — Configure memory parameters
```

## Permissions

Permissions control tool access: allow, deny, or require approval.

Canonical permission keys are tool names such as `edit`, `read`, and `bash`. Use `edit` for file modification rules; legacy agent `tools.write`, `tools.patch`, and `tools.multiedit` booleans are converted to `permission.edit`.

### Permission actions

| Action | Meaning |
|---|---:|
| `"allow"` | Execute without user approval |
| `"ask"` | User must approve each invocation |
| `"deny"` | Blocked entirely |

### Top-level permissions (all agents)

```jsonc
{
  "permission": {
    "read": "allow",
    "edit": "allow",
    "glob": "allow",
    "grep": "allow",
    "list": "allow",
    "bash": "ask",
    "task": "allow",
    "webfetch": "allow",
    "websearch": "allow",
    "external_directory": "deny",
    "download": "allow",
    "lsp": "allow",
    "doom_loop": "allow",
    "dagwrite": "allow",
    "dagread": "allow",
    "question": "allow",
    "todowrite": "allow",
    "todoread": "allow",
    "skill": "allow"
  }
}
```

### Per-agent permissions

Override in agent config:
```jsonc
{
  "agent": {
    "safe-agent": {
      "permission": { "bash": "deny", "edit": "deny" }
    }
  }
}
```

Agent permissions merge with and override top-level permissions.

### Path-scoped permissions

Some tools support pattern-based rules:

```jsonc
{
  "permission": {
    "bash": {
      "*": "ask",
      "git *": "allow",
      "npm test": "allow",
      "rm *": "deny"
    },
    "edit": {
      "*": "ask",
      "src/**": "allow",
      "*.config.*": "deny"
    },
    "read": {
      "*": "allow",
      ".env*": "deny"
    }
  }
}
```

`*` is the default rule. More specific patterns override it. Rules are evaluated in order — first matching rule wins.

### Shorthand

Top-level shorthand for all tools:
```jsonc
{ "permission": "allow" }
```

Equivalent to setting all tools to `"allow"`.

### Control profiles

Control profiles set the overall approval, sandbox, and network posture. Agents inherit a profile and can override it.

| Profile | Approval summary | Sandbox | Network |
|---|---|---|---|
| `guarded` | Low-risk allow; many medium/high-risk actions ask; hardline shell is denied | `workspace_write` | `restricted` |
| `autonomous` | Low/medium-risk allow; high-risk deny unless explicitly allowed by the profile; asks are converted to denies when unattended | `workspace_write` | `restricted` |
| `full_access` | Silent allow of every capability, including protected paths, secrets, destructive shell, external writes, identity/channel actions, and plugin/platform operations | `none` | `enabled` |

Per-agent override:
```jsonc
{
  "agent": {
    "my-agent": {
      "controlProfile": "autonomous"
    }
  }
}
```

### SmartAllow

SmartAllow reduces prompt frequency by auto-approving certain operations without user interaction.

- **Safe eligible cases** — low-risk reads, known-safe patterns, non-destructive operations.
- **Redacted evidence** — for secret-like files (`.env`, credentials), metadata or redacted evidence is used instead of raw content.
- **Stricter autonomous threshold** — in `autonomous` profile, SmartAllow applies a higher confidence bar. Cases below the threshold are auto-denied (not prompted).
- **Circuit breaker** — after three high-confidence disagreements between the classifier and user feedback, SmartAllow disables itself for that session and falls back to normal policy handling.

### Common permission patterns

**Read-only agent** — can read and search but never modify:

```jsonc
{ "permission": { "edit": "deny", "bash": "deny" } }
```

**Sandboxed execution** — bash allowed only for specific safe commands:

```jsonc
{ "permission": { "bash": { "*": "ask", "git status": "allow", "npm test": "allow", "bun test": "allow" } } }
```

**Protect sensitive files** — read everything except secrets:

```jsonc
{ "permission": { "read": { "*": "allow", ".env*": "deny", "*credentials*": "deny", "*.pem": "deny" } } }
```

**Full trust** — allow everything without asking:

```jsonc
{ "permission": "allow" }
```

**Source code only** — only edit source files, not config:

```jsonc
{ "permission": { "edit": { "*": "ask", "src/**": "allow", "test/**": "allow" } } }
```

### Troubleshooting

- **Agent getting "permission denied"** — check both top-level and agent-level permissions; agent-level overrides top-level; check path-scoped rules (more specific patterns override `*`)
- **Permission change not taking effect** — reload with `runtime_reload(target: "config")`; permission changes cascade to agent reload automatically
- **"ask" prompts too frequent** — consider path-scoped rules to auto-allow safe patterns; use `*` as default with specific overrides
- **Memory/library not working** — check `library.autonomy` is not `false`; check `library.memory.enabled` is not `false`; check `library.experience.encode` / `library.experience.retrieve` are not `false`
- **Embedding/rerank errors** — verify API key and baseURL in top-level `embedding` / `rerank` sections; check the model name matches what the provider expects; embedding failures are logged at startup

## Reload

```
runtime_reload(target: "config")
```

Library and permission changes cascade to agent reload automatically.
