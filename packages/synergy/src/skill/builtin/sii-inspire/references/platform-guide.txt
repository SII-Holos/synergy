# Platform Guide — SII 启智平台

## Workspace Types & Network

| Space | Internet | GPU | Use Case |
|-------|----------|-----|----------|
| 可上网GPU资源 | ✅ whitelist | 4090 (48GB) | Download data, set up environments, light training |
| 分布式训练空间 | ❌ | H100 (80GB), H200 (141GB) | Large-scale training. Images and data must be prepared in advance |

Bandwidth: 10Gbps shared outbound for all internet-enabled spaces.

## Storage Architecture

```
/inspire/hdd/project/{project_en_name}/        ← HDD, shared across ALL spaces in same project
/inspire/ssd/{workspace_id}/{project_en_name}/  ← SSD, shared across spaces
/inspire/hdd/global_user/{username}/            ← Personal global directory
/inspire/hdd/project/{project_en_name}/public   ← Project-level shared directory for team members
```

- Same project = same storage, regardless of which space you're in
- Different projects cannot see each other's storage
- Deletion is IRREVERSIBLE (GPFS has no recycle bin)
- Check quota: `df -h /inspire/hdd/project/{name}/`
- Expand quota: project owner requests via Feishu approval or contacts advisor

## Priority & Scheduling

- Each project has a max priority (visible in `inspire_status` as `priority_name`)
- Priority 1-3: can be preempted (killed) by higher-priority tasks
- Priority ≥4: will NOT be preempted
- Same priority: tasks using fewer GPUs are scheduled first
- Multi-node tasks: all nodes must be simultaneously available
- GPU utilization too low → auto-reclaimed (policy varies by space)

## Budget (点券)

- 导师项目/标杆项目: refreshed quarterly (Q1=Jan-Mar, Q2=Apr-Jun, etc.)
- 公共科研项目: 672 points per user per week (Mondays ~10:00)
- Budget ≤0: may block task creation. Workaround: use priority 1-3 for CPU tasks (exempt from budget)
- Max GPU quota per project: 32 cards (导师项目, may be adjusted)

## Image Management

Two independent registries (NOT interchangeable):
- 七宝: `docker-qb.sii.edu.cn` (main campus)
- 松江: `docker.sii.shaipower.online`

Push workflow:
1. `docker login docker-qb.sii.edu.cn` (credentials from platform's Image Management → Local Push page)
2. `docker tag <local> docker-qb.sii.edu.cn/inspire-studio/<name>:<tag>`
3. `docker push docker-qb.sii.edu.cn/inspire-studio/<name>:<tag>`
4. First push: claim the image in platform's Image Management page

Image saves:
- Only saves container filesystem changes (`/usr/local`, `/root`, etc.)
- Does NOT save mounted storage (`/inspire/hdd/...`)
- Clean caches before saving: `apt clean && pip cache purge && rm -rf ~/.cache`
- Images >100GB may fail to save
