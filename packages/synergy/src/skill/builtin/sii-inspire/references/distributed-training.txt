# Distributed Training — SII 启智平台

## Environment Variables (Auto-Injected)

The platform automatically injects these into every training job container:

| Variable | Description |
|----------|-------------|
| `MASTER_ADDR` / `PET_MASTER_ADDR` | Master node address |
| `MASTER_PORT` / `PET_MASTER_PORT` | Master node port |
| `PET_NNODES` | Total number of nodes |
| `PET_NODE_RANK` | Current node rank (0-indexed) |
| `PET_NPROC_PER_NODE` | GPUs per node |

Do NOT hardcode these. Use the variables directly in your launch command.

## Launch Command Templates

### torchrun (recommended)
```bash
torchrun \
    --nnodes ${PET_NNODES} \
    --node_rank ${PET_NODE_RANK} \
    --nproc_per_node ${PET_NPROC_PER_NODE} \
    --master_addr ${MASTER_ADDR} \
    --master_port ${MASTER_PORT} \
    train.py --config config.yaml
```

### DeepSpeed
```bash
deepspeed \
    --num_nodes ${PET_NNODES} \
    --num_gpus ${PET_NPROC_PER_NODE} \
    --master_addr ${MASTER_ADDR} \
    --master_port ${MASTER_PORT} \
    --node_rank ${PET_NODE_RANK} \
    train.py --deepspeed ds_config.json
```

### Megatron-LM
```bash
python -m torch.distributed.launch \
    --nproc_per_node ${PET_NPROC_PER_NODE} \
    --nnodes ${PET_NNODES} \
    --node_rank ${PET_NODE_RANK} \
    --master_addr ${MASTER_ADDR} \
    --master_port ${MASTER_PORT} \
    pretrain_gpt.py
```

Note: for torchrun, do NOT pass `--rdzv_id` or `--rdzv_backend` — the platform handles rendezvous.

## NCCL Troubleshooting

### Shared memory error
Error: `Error while creating shared memory segment` or `/dev/shm` related errors.

Fix: set shared memory when creating the task. Use `shm` parameter in `inspire_submit` (default 1200 MB). For multi-GPU training, recommend ≥64GB.

### NCCL timeout / Socket timeout / Watchdog timeout
Add these environment variables to the beginning of your command:
```bash
export NCCL_P2P_DISABLE=1
export NCCL_NVLS_ENABLE=0
export NCCL_IB_RETRY_CNT=255
export NCCL_IB_TIMEOUT=25
export MASTER_ADDR=${PET_MASTER_ADDR}
export MASTER_PORT=${PET_MASTER_PORT}
```

If still failing: likely hardware fault on the node. Record task ID + node name and contact ops.

### H100 multi-node training slow
Check if InfiniBand drivers are installed:
```bash
apt install libibverbs1 libibverbs-dev ibverbs-providers ibverbs-utils librdmacm1 librdmacm-dev rdmacm-utils
```

### UCX library conflict (`libucc.so.1: undefined symbol`)
```bash
export LD_LIBRARY_PATH="/opt/hpcx/ucc/lib:/opt/hpcx/ucx/lib:$LD_LIBRARY_PATH"
export UCC_TLS="^sharp"
```

## Hardware Reference

| GPU | Memory | Cluster | Network |
|-----|--------|---------|---------|
| H100 SXM | 80GB | 七宝 | RoCE, 400Gbps × 8 link |
| H200 SXM | 141GB | 七宝 (3 机房) | RoCE, 400Gbps × 8 link |
| 4090 | 48GB | 七宝 (可上网区) | Standard Ethernet |
| Ascend 910B | 64GB | 松江 | — |

- H200 does NOT have CUDA 12.9 (only 12.8 as of 2026-03)
- Cross-node NVLink (MNNVL) is NOT supported; cross-node communication uses RDMA/RoCE
- NVLink is intra-node only

## Fault Tolerance

Enable auto fault tolerance in `inspire_submit` by adding to the job config:
```
auto_fault_tolerance: true
```

This allows the platform to automatically restart the task on a different node if the current node fails.

## HPC / Slurm Jobs

- HPC spaces use Slurm scheduling (NOT the platform's native scheduler)
- Must use images with `slurm` in the name
- Slurm `MaxStepCount` cannot be user-adjusted — split into multiple smaller jobs if needed
- Environment: non-interactive shell, same conda init rules apply
