# SII 启智平台 — Inspire Tools Guide

Use this skill when the user wants to run experiments, manage Docker images, or check GPU resources on the SII 启智 AI platform.

## First-Time Setup

If inspire_config shows no defaults configured yet:

1. Run `inspire_status` to discover projects, workspaces, and available GPUs
2. Set defaults for the user's primary project:
   ```
   inspire_config(action="set", key="defaultProject", value="项目名")
   inspire_config(action="set", key="defaultWorkspace", value="分布式训练空间")
   inspire_config(action="set", key="defaultComputeGroup", value="cuda12.8版本H100")
   inspire_config(action="set", key="defaultImage", value="docker-qb.sii.edu.cn/inspire-studio/xxx:tag")
   inspire_config(action="set", key="defaultPriority", value="9")
   inspire_config(action="set", key="defaultShm", value="1200")
   inspire_config(action="set", key="commandPrefix", value="source /opt/conda/etc/profile.d/conda.sh && conda activate myenv && cd /inspire/hdd/project/{en_name}/code")
   ```
3. After setup, `inspire_submit` only needs `name` and `command`

## Core Workflows

### Submit a GPU training job

```
inspire_status(project="...", workspace="分布式训练空间")  → check available GPUs
inspire_submit(name="exp-001", command="python train.py --config configs/baseline.yaml")
inspire_jobs(status="running")                            → monitor
inspire_job_detail(job_id="...")                          → check details if failed
```

### Submit an HPC (Slurm) job

For CPU-intensive tasks like data preprocessing, evaluation, or file operations:

```
inspire_submit_hpc(name="preprocess-data", entrypoint="python preprocess.py",
  workspace="高性能计算", number_of_tasks=2, cpus_per_task=4, memory_per_cpu="8G")
inspire_job_detail(job_id="hpc-job-xxx")                  → HPC jobs use hpc-job- prefix
```

### Deploy an inference service

Deploy a trained model as a serving endpoint:

```
inspire_inference(action="create", name="llama3-serving",
  model_id="model-xxx", model_version=1, command="python -m vllm.entrypoint.openai.api_server",
  image="docker-qb.sii.edu.cn/inspire-studio/vllm:latest", port=2400, replicas=1)
inspire_inference(action="detail", serving_id="sv-xxx")   → inference servings use sv- prefix
inspire_inference(action="stop", serving_id="sv-xxx")
```

### Prepare a Docker image

```
inspire_images(search="torch")                            → find existing images
inspire_images(repo="dhyu-wan-torch29")                   → check versions
inspire_image_push(image="my-train:v1", description="PyTorch 2.9 + CUDA 12.8")  → push new image
```

### Download data for offline training

Offline spaces (分布式训练空间, 高性能计算) have NO internet. To use HuggingFace models or external data:

1. Submit a CPU job in an online space to download:
   ```
   inspire_submit(name="download-model", workspace="CPU资源空间",
     command="export HF_ENDPOINT=https://hf-mirror.com && huggingface-cli download meta-llama/Llama-3-8B --local-dir /inspire/hdd/project/{en_name}/models/llama3-8b")
   ```
2. Wait for download to complete (same project directory is shared across all spaces)
3. Submit training job in offline space referencing the downloaded path

### Capture training output

The platform has no log API. Always redirect output to a file for debugging:

```
inspire_submit(name="exp-001",
  command="python train.py --config baseline.yaml 2>&1 | tee /inspire/hdd/project/{en_name}/logs/exp-001.log")
```

To read results after completion, submit a lightweight CPU job:
```
inspire_submit(name="read-results", workspace="CPU资源空间",
  command="cat /inspire/hdd/project/{en_name}/logs/exp-001.log && cat /inspire/hdd/project/{en_name}/results/*.json")
```

### Set up monitoring with Agenda

After submitting jobs, create an agenda watch to get notified when they finish:

```
agenda_create({
  title: "Monitor: exp-001",
  prompt: "Check inspire_jobs(status='running'). Report when exp-001 completes or fails.",
  triggers: [{ type: "watch", watch: { kind: "tool", tool: "inspire_jobs", args: { status: "running" }, interval: "5m", trigger: "change" } }]
})
```

## Key Platform Constraints

- **Offline spaces**: No internet. No pip install, git clone, wget in commands. All dependencies must be in the Docker image. All data must be in the project directory.
- **Non-interactive shell**: ~/.bashrc is NOT loaded. Must source conda manually. Use `commandPrefix` to automate this.
- **Priority ≥4**: Won't be preempted. Priority 1-3 can be killed by higher-priority tasks.
- **Two registries**: 七宝 uses `docker-qb.sii.edu.cn`, 松江 uses `docker.sii.shaipower.online`. They are NOT interchangeable.
- **Storage is shared**: `/inspire/hdd/project/{en_name}/` is accessible from ALL spaces in the same project.
- **No log API**: Always use `2>&1 | tee` to capture output to a file in the project directory.
- **Task types by ID prefix**: `job-xxx` (GPU training), `hpc-job-xxx` (HPC/Slurm), `sv-xxx` (inference serving). The `inspire_stop` and `inspire_job_detail` tools auto-detect the type by prefix.
- **OpenAPI required**: Task submission and stop operations use the official OpenAPI. HPC submission falls back to cookie API if OpenAPI is unavailable. Inference serving creation requires OpenAPI.
