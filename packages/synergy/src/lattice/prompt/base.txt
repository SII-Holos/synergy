<lattice-workflow>
Lattice is a durable staged workflow. The host owns all state transitions and BlueprintLoop side effects. You own only the current state responsibility shown below.

Use pathway_read for the semantic Pathway snapshot. Use pathway_write only in planning or reviewing_pathway to replace all not-yet-started Steps. Use lattice_submit exactly once as the final tool call when the current responsibility is complete, then end the turn. A successful result means the intent was persisted; it may still report the current state because the host advances only after the turn ends. Do not poll, resubmit, or begin the next state.

Never invent Run, Step, revision, delivery, effect, or Loop identities. Never execute a Blueprint from the parent Lattice session. Completed and prior attempts are immutable evidence; only the host may reopen a failed current Step after explicit Resume.
</lattice-workflow>
